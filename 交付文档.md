## 一、项目概述

DietAI 是一款智能饮食健康管理应用，核心能力包括 AI 食物识别与营养分析、个性化饮食建议、健康目标追踪、养生饮食推荐。Milestone 1 聚焦四大核心功能的完整实现。

### 技术架构

| 层级 | 技术栈 |
| --- | --- |
| 前端 | Flutter (Dart)，支持 Android / iOS / Windows |
| 后端 | FastAPI (Python)，PostgreSQL + Redis + MinIO |
| AI Agent | LangGraph + ChromaDB (RAG)，通义千问视觉模型 + DeepSeek 大语言模型 |
| 定时任务 | APScheduler |
| 对象存储 | MinIO |

## 六、AI Agent 架构

### 双模型架构

| 用途 | 模型 | 厂商 | 说明 |
| --- | --- | --- | --- |
| 视觉识别 | qwen-vl-max | 阿里通义千问 | 食物图片识别 |
| 分析推理 | deepseek-v4-flash | DeepSeek | 营养分析、建议生成、过敏检查 |

### Agent 流程

```
用户上传图片 → qwen-vl-max 识别食物 → deepseek-v4-flash 提取营养素
→ RAG 检索相关知识 → 过敏交叉检查 → 生成个性化建议 → SSE 流式返回前端
```

### RAG 优化

- **Metadata Filtering**: 按人群标签(crowd_减脂/crowd_健身)、体质(constitution)、节气(solar_term)、季节(season) 多维度过滤
- **分区缓存**: RAGCache 类，基于 query+filter 哈希缓存，TTL 30分钟，按人群标签自动分区
- **缓存失效**: 知识库更新时调用 rebuild_vector_store() 自动清除缓存并重建索引

## 九、启动与部署

### 环境依赖

- Python 3.12+ (uv 管理)
- Flutter 3.x
- Docker Desktop (MinIO)
- PostgreSQL + Redis

### 启动步骤

```bash
# 1. 启动 MinIO
docker compose up -d minio

# 2. 启动后端
cd DietAI
uv run python main.py

# 3. 启动 LangGraph Agent 服务
uv run langgraph dev --port 2024

# 4. 启动前端
cd frontend_flutter
flutter run -d windows
```

### 环境变量 (.env)

| 变量 | 说明 |
| --- | --- |
| DATABASE_URL | PostgreSQL 连接串 |
| REDIS_URL | Redis 连接串 |
| MINIO_ENDPOINT | MinIO 地址 |
| MINIO_ACCESS_KEY / SECRET_KEY | MinIO 凭证 |
| DASHSCOPE_API_KEY | 通义千问 API Key（视觉模型） |
| DEEPSEEK_API_KEY | DeepSeek API Key（分析模型） |
| JWT_SECRET | JWT 签名密钥 |
