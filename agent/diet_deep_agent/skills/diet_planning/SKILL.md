# 膳食规划技能

## 触发条件
- 用户问"吃什么"、"推荐一下"
- 用户要膳食建议、食谱推荐
- 用户问某一餐应该怎么吃

## 执行流程

### 1. 获取剩余配额
- 使用 `get_daily_status` 获取今日剩余热量和宏量营养素配额

### 2. 检索营养知识
- 使用 `query_nutrition_knowledge` 检索适合的食物选项

### 3. 个性化过滤
- 读取 `/memories/preferences.md` 获取用户偏好
- 读取 `/memories/profile.md` 获取过敏原和饮食限制
- 排除用户不喜欢的食物和过敏原
- 优先推荐用户喜欢的食物

### 4. 生成方案
- 生成 2-3 个具体的膳食方案
- 每个方案包含：食物列表、预估营养值、配额匹配度
- 标注哪个方案最能补充当前营养缺口

## 输出格式
```
### 🍽️ 今日推荐

**剩余配额**: X kcal | 蛋白质 Xg | 碳水 Xg | 脂肪 Xg

#### 方案一：[名称]
- [食物1] × [份量]
- [食物2] × [份量]
- 预估：X kcal | 匹配度 X%

#### 方案二：[名称]
...

💡 推荐方案X，因为[原因]
```
