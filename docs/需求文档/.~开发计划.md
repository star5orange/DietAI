## 六、风险预警

| 风险 | 概率 | 影响 | 应对 |
| --- | --- | --- | --- |
| 某人进度滞后 | 高 | 影响联调 | 每日站会跟踪，滞后1天以上统筹介入协调 |
| 前后端接口理解不一致 | 高 | 联调返工 | 接口文档三方确认，联调前先跑通 Swagger |
| AI 建议质量不达标 | 中 | 用户体验差 | 第2周前完成质量验证，预留优化时间 |
| 前端人员刚接手不熟悉 | 高 | 前端进度慢 | 统筹协助梳理现有代码，第1周重点熟悉项目 |
| 本地通知兼容性问题 | 低 | 部分设备收不到 | 优先适配主流设备，记录兼容问题 |
| 养生知识科学性存疑 | 中 | 误导用户 | 知识库需人工审核，AI 输出加免责声明 |

## 七、优先级说明

| 优先级 | 含义 | 要求 |
| --- | --- | --- |
| P0 | 核心功能，必须完成 | 阻塞验收，不可延期 |
| P1 | 重要功能，应当完成 | 影响体验，尽量完成 |
| P2 | 锦上添花，可延后 | 不影响核心流程，可推到 Milestone 2 |

> **原则**: P0 全部完成 → P1 尽量完成 → P2 视进度决定。任何 P0 任务延期需立即上报统筹。
